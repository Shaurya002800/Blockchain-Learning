================================================================================
BLOCKCHAIN PROJECT DEVELOPMENT: PART 4 - DEFI TOKEN + SWAP
================================================================================

OVERVIEW
--------------------------------------------------------------------------------
This project builds a foundational decentralized finance (DeFi) ecosystem 
consisting of a custom token and a protocol to trade it.

THE GOAL:
Hold Token -> Transfer Token -> Trade Token -> Swap for Another Token


1. THE ERC-20 TOKEN ("Your Coin")
--------------------------------------------------------------------------------
Standard: ERC-20 (Fungible, interchangeable, and tradable).
Real-World Examples: USDC, UNI, LINK.

KEY CONTRACT FEATURES:
- MAX_SUPPLY:    Hard cap on total tokens (e.g., 1,000,000).
- mint():        Function to create new tokens (usually owner-only).
- burn():        Reduces supply to increase scarcity.
- Decimals:      Standard is 18 (e.g., 1 token = 10^18 internal units).


2. THE AMM SWAP POOL ("The Automated Exchange")
--------------------------------------------------------------------------------
This allows trading without a bank or broker using math-based prices.

THE AMM FORMULA: [ x * y = k ]
- Prices adjust automatically based on supply.
- If one token becomes rare, its price increases instantly.

LIQUIDITY POOLS (LP):
- A storage pool containing two tokens (Token A + Token B).
- Liquidity Providers (LPs) deposit these tokens and earn swap fees in return.


3. THE TECHNICAL STACK
--------------------------------------------------------------------------------
- Smart Contracts: Solidity + OpenZeppelin (Secure templates).
- Environment:     Hardhat (Compiling/Testing).
- Frontend:        Next.js + wagmi + RainbowKit (UI & Wallet connection).
- Logic Library:   ethers.js (Blockchain communication).


4. PROJECT WORKFLOW & COMMANDS
--------------------------------------------------------------------------------
DEPLOYMENT FLOW:
1. Deploy Token Contract -> 2. Deploy Swap Contract -> 3. Add Initial Liquidity.

MINTING & APPROVAL:
- approve(): Users must authorize the swap contract to spend their tokens 
             before a trade can occur. (Analogy: Giving a check to a cashier).

SWAP FLOW:
Connect Wallet -> Input Amount -> Approve Token -> Confirm Swap -> Get New Token.


5. DEFI SECURITY & RISKS (CRITICAL)
--------------------------------------------------------------------------------
DeFi handles real capital, making safety the #1 priority.

MAJOR ATTACK VECTORS:
- Reentrancy:   Recursive drainage of funds. (Use: ReentrancyGuard).
- Slippage:     Bad pricing during high volatility. (Use: Slippage protection).
- Flash Loans:  Using massive, uncollateralized loans to manipulate markets.
- Oracle Hacks: Manipulating price feeds to steal assets.

--------------------------------------------------------------------------------
FINAL MENTAL MODEL:
- ERC-20 Token     = The money
- AMM Pool         = The automatic exchange booth
- Liquidity Pool   = The token storage
- Smart Contract   = The financial rules/logic
- Approval         = Granting permission to trade
- Blockchain       = The permanent, public financial record
