OVERVIEW
--------------------------------------------------------------------------------
This section covers the architecture of NFT (Non-Fungible Token) collections, 
moving from basic concepts to a real-world development workflow.

CORE CONCEPT:
- NFT = Unique Digital Ownership.
- Unlike coins, NFTs are non-interchangeable and represent specific assets.
- Blockchain stores: Ownership, Token ID, and Metadata Reference.
- IPFS/Arweave stores: The actual large files (Images/Videos).


1. THE NFT STANDARDS & STORAGE
--------------------------------------------------------------------------------
ERC-721 ("The NFT Standard"):
- The global rulebook for NFTs on Ethereum/EVM chains.
- Handles ownership, transfer logic, and marketplace compatibility.
- Without this, OpenSea or MetaMask wouldn't know how to display your NFT.

IPFS ("The File Warehouse"):
- Storage for artwork and metadata because blockchain is too expensive.
- FLOW: Artwork -> Upload to IPFS -> Get Hash/Link -> Store Link in Contract.

OPENSEA:
- The "Amazon/eBay" for NFTs. It automatically reads ERC-721 contracts.


2. THE DEVELOPMENT WORKSPACE
--------------------------------------------------------------------------------
Before coding, developers prepare these essential tools:

- Node.js:          Powers the JS ecosystem and Hardhat.
- VS Code:          The primary editor for Solidity and React.
- MetaMask:         User wallet. (Rule: NEVER share your 12-word seed phrase).
- Hardhat:          The blockchain testing laboratory.
- OpenZeppelin:     Provides secure, audited "templates" for ERC-721 logic.


3. PROJECT INITIALIZATION COMMANDS
--------------------------------------------------------------------------------
- mkdir my-nft          -> Create folder
- npm init -y           -> Initialize package.json (Dependency manager)
- npm install hardhat   -> Add dev environment
- npx hardhat init      -> Setup folders and sample contracts


4. THE NFT LIFECYCLE (MINTING)
--------------------------------------------------------------------------------
MINTING: The act of creating a new token on the blockchain.
- Blockchain creates the token.
- Ownership is assigned to the user's wallet.
- The asset becomes permanent.

COMPLETE PROJECT FLOW:
Create Contract -> Upload to IPFS -> Deploy -> Build UI -> User Mints -> Done.


5. RECOMMENDED BEGINNER NFT STACK
--------------------------------------------------------------------------------
- Network:         Polygon (Cheap & Fast)
- Standard:        ERC-721 (OpenZeppelin)
- Storage:         IPFS via Pinata
- Frontend:        Next.js
- Integration:     wagmi + RainbowKit

--------------------------------------------------------------------------------
FINAL MENTAL MODEL:
- NFT                = Unique ownership token
- ERC-721            = The rulebook
- IPFS               = The image/file storage
- Smart Contract     = The logic for minting
- Wallet             = Your identity/signature
- Frontend           = The website interface
- Blockchain         = The permanent ledger of truth
